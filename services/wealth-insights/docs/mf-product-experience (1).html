02First-Time Experience — Revelation Sequence
XIRR is not the hook. An abstract percentage doesn't land for retail Indian investors — they don't know if 14.2% is good or bad without context. The hook needs to answer a question they already have a felt answer for: "Am I better off than the bank?" / "Am I being ripped off?" / "What did I miss?" Those three land because users have anchors for them.
1
Visible Analysis Screen
Not a spinner. Real progress: "Found 12 funds across 4 folios… Tracking 47 SIP installments… Calculating tax implications…" Builds anticipation, communicates depth.
2
The Real Hook — A Relatable Comparison
Choose the most emotionally resonant frame based on portfolio composition. Priority order:
If Regular Plans: "Your distributor earned ₹X from your money" If underperforms FD: "An FD would've given you ₹X more" If outperforms: "You beat a fixed deposit by ₹X,XXX" Always available: "Your money has been working for N days"
The distributor framing is especially powerful — it's personal, it's loss framing, it's often a shock. "₹12,400 went to someone else" lands very differently from "your XIRR is 14.2%."
3
Portfolio Fitness Score
One number out of 100 with a label and the single biggest drag. "Your portfolio scores 58/100. Your biggest drag: Regular plans (−22 points)." This becomes the user's personal benchmark to improve.
portfolioFitnessScore (composite — defined in §7)
4
Insight Unlock — N Findings
List of insight teasers, grouped by category. Not answers — questions. "2 of your funds secretly hold 47% of the same stocks." "One investment is costing you ₹X/year in unnecessary fees." Teasers create urgency to explore.
Ranked by ₹ impact5 categories shownQuestions, not summaries
5
Single CTA — Highest-Priority Actionable
One button. Specific to their data. The actionable with the highest ₹ impact × lowest friction. If Regular plans: "Switch 6 funds to Direct — save ₹58,000 over 10 years." If ELSS imminent: "Your ELSS unlocks in 30 days. Plan your tax-free window."
03Frequency Model — Reworked
Many insights are not recurring — they fire once, when a condition is first met. The engine re-evaluates on every data refresh, but the user only sees a card when something meaningfully changes. This prevents the feed from feeling repetitive and keeps every card feeling fresh.

05Card Journey Examples

Universal 5-Card Structure

Card 1 — AnswerThe verdict. Pass / Warning / Fail. One number or signal, large and clear.
Card 2 — EducationWhat is this metric? Why does it matter? 2–3 sentences max. No jargon without explanation.
Card 3 — Your DataUser's specific situation — their actual funds and numbers. Not generic.
Card 4 — ImpactWhat this costs or saves in ₹. The number that makes it real.
Card 5 — Action3 options always. Last option always: "Ask about this" — opens AI chat with full context pre-loaded.

06Actionables — Fixed + Dynamic

A given user will have 2–4 relevant fixed actionables based on their cohort. The dynamic LLM layer activates only when no fixed actionable scores above a relevance threshold.

Fixed Actionables
🔄
Switch Regular Plans to Direct When: any regular plan
Lists each fund with exact extra TER, annual drag in ₹, 10-year compounding impact. Sorted by annual drag. For ELSS on regular, flags lock-in — notes earliest switch date.
terAnalysis + direct-vs-regular + taxHarvesting.daysToLTCG
📅
ELSS Unlock Calendar Always Triggered at 30d
Per-installment unlock dates. Unrealised gain per lot. Estimated LTCG tax at redemption. STCG vs LTCG status per lot. CTA: "Set reminder" per unlock.
taxHarvesting (FIFO, per-lot daysToLTCG)
✂️
Tax Harvest Window Jan–Mar + LTCG within ₹1.25L available
Exact funds + exact lots to redeem to use up LTCG exemption. Suggests what to reinvest in (same fund, different folio) to reset cost basis. Only fires when the numbers are real and meaningful.
taxHarvesting.exemptionUsed + year-end window
🧹
Consolidate Overlapping Funds When: overlap >40%
Names the specific pair. Shows top 5 common holdings. Recommends which to exit based on TER + trailing performance vs category. Suggests where to redirect proceeds.
overlapAnalysis + terAnalysis + benchmarkComparison
⏸️
Revive or Exit Dormant Holding 6+ months no activity
Per dormant folio: current XIRR vs category average. Three-way decision: top up (performing), switch (underperforming), or exit. Dormant SIPs flagged separately from lumpsum holdings.
sipAnalysis.missedMonths + xirrAnalysis + benchmarkComparison
📈
Add to Best Performer During Dip Market drop >5% + fund still outperforming
Identifies user's best fund (3yr XIRR vs category) that has dropped this week but remains a category leader. "This fund has recovered from every dip in 3 years. This might be the time to add, not exit." Behavioral counter-signal to panic.
benchmarkComparison + marketEventTrigger + behavioralAnalyser
🏷️
Redirect Micro Holdings Holdings <₹1,000 detected
Micro holdings add portfolio noise without meaningful diversification. Lists them, shows combined value, suggests: consolidate into main holding or exit. Not judgmental — surfaces clutter cleanly.
anomalyDetector: micro holdings
⚖️
Rebalance After Drift Monthly: if drift >10%
When equity/debt or large/mid/small cap split drifts >10% from original allocation (inferred from transaction history), shows current vs original split and a rebalancing suggestion. Persists monthly while drift remains.
assetAllocation + marketCapAllocation + cashflowAnalysis
🔔
Set SIP Increase Reminder When: SIP unchanged 12+ months + salary month detected
Detects if SIP amount hasn't changed in 12+ months. If salary credit pattern is visible in cashflow, suggests a SIP step-up. Shows impact: "Increasing SIP by ₹1,000/month adds ₹X in 10 years." Not pushy — framed as an opportunity.
sipAnalysis + cashflowAnalysis + xirrAnalysis (projection)
✦ Dynamic Actionable — LLM-Generated
🤖
Dynamic Actionable
When no fixed actionable scores above relevance threshold (default: 60/100), the engine sends the full portfolio analysis JSON to an LLM with the system prompt below, asking for one new actionable not duplicating any existing one. Output is schema-validated before rendering — falls back to next highest fixed actionable if invalid.
All 16 analysis modules + relevance scores of all fixed actionables
System Prompt — Dynamic Actionable Generator
You are a SEBI-registered investment advisor helping an Indian retail mutual fund investor.

You will be given a structured JSON summary of the user's portfolio analysis, including:
- portfolioSummary, xirrAnalysis (per-folio + aggregate), cashflowAnalysis, sipAnalysis
- taxHarvesting (per lot, FIFO), overlapAnalysis, benchmarkComparison, riskMetrics
- behavioralAnalyser signals, 8 what-if scenario results with relevance scores
- List of fixed actionables already scored or shown

Your task: Generate ONE new actionable that is:
1. Specific to this user's actual data — not generic advice
2. Not a duplicate of any fixed actionable provided in the input
3. Written in clear, plain language (Hinglish is acceptable, avoid unexplained jargon)
4. Quantified where possible — show ₹ impact, not percentages alone
5. Actionable in 1 interaction — user can act without leaving the app

// Guardrails: Never generate tax-specific redemption instructions. Never speculate on future NAV. Never compare to specific stocks or crypto. Stick to portfolio structuring, behavioral, or fee-efficiency insights.

// Output ONLY valid JSON. No preamble, no markdown, no explanation. Schema:
{
  "title": "short imperative title (max 8 words)",
  "trigger": "the data point that triggered this",
  "description": "2–3 sentences, user-specific, ₹-quantified where possible",
  "ctaPrimary": "action button label",
  "ctaSecondary": ["label 1", "label 2"],
  "dataPoints": ["module.field used", "module.field used"]
}

🔇 Suppression Rules

Max 1 non-critical push per day per user
P0 triggers (ELSS, tax boundary) always bypass daily limit
Market event push only if user holds affected fund category
Same what-if scenario not shown twice within 4 weeks
